# RichEditor 改善提案

## 現在の問題点

### 1. パフォーマンス関連

#### 1.1 入力パフォーマンス
- **問題**: `handleInput` 関数で 300ms の debounce がかかっているが、大きな記事を入力すると重い
- **影響**: 長文記事で入力遅延が発生
- **改善案**:
  - Virtual DOM の採用（Svelte 5 の runes を活用）
  - 入力履歴の保存ロジックを Web Worker に移管
  - 不要な `innerHTML` の更新を削減

#### 1.2 メモリリーク
- **問題**: `history` 配列が無限に増加する可能性（`MAX_HISTORY_ENTRY_SIZE` チェックはあるが）
- **改善案**:
  - 履歴の数を制限（例：最大 50 件）
  - 古い履歴を間引くロジックを追加
  - WeakMap を使用した参照管理

### 2. UX/UI 関連

#### 2.1 キーボードショートカット
- **現状**: `Ctrl/Cmd + S` で保存のみ
- **改善案**:
  ```
  Ctrl/Cmd + B: 太字
  Ctrl/Cmd + I: イタリック
  Ctrl/Cmd + U: 下線
  Ctrl/Cmd + K: リンク挿入
  Ctrl/Cmd + Z: アンドゥ
  Ctrl/Cmd + Y: リドゥ
  Ctrl/Cmd + H: 見出し
  Ctrl/Cmd + 1-6: 見出しレベル 1-6
  Ctrl/Cmd + Shift + L: リスト
  Ctrl/Cmd + Shift + N: 番号付きリスト
  Ctrl/Cmd + Shift + Q: 引用
  Ctrl/Cmd + Shift + C: コードブロック
  Ctrl/Cmd + Shift + I: 画像挿入
  Ctrl/Cmd + Shift + T: テーブル挿入
  Alt + ↑/↓: 要素の移動
  Esc: スラッシュメニューを閉じる
  ```

#### 2.2 アクセシビリティ
- **問題**:
  - ARIA 属性が不足
  - キーボードナビゲーションが不完全
  - スクリーンリーダー対応が不十分
- **改善案**:
  - `role="toolbar"` をツールバーに追加
  - 各ボタンに `aria-label` を設定
  - フォーカストラップの実装
  - スラッシュメニューに `role="menu"` を追加

#### 2.3 モバイル対応
- **問題**:
  - スマホでツールバーが画面からはみ出る
  - タッチ操作での要素選択が困難
  - リサイズハンドルが小さすぎる
- **改善案**:
  - レスポンシブツールバー（折りたたみ式）
  - タッチジェスチャーのサポート（スワイプで削除など）
  - モバイル用クイックツールバー

### 3. 機能関連

#### 3.1 共同編集機能
- **提案**: WebSocket を使用したリアルタイム共同編集
  - Operational Transformation (OT) または CRDT の実装
  - 編集中のユーザー表示
  - 変更履歴の可視化

#### 3.2 AI 機能
- **提案**:
  - 文章校正サポート（文法チェック）
  - 自動要約生成
  - SEO 推奨キーワード提案
  - 見出しの自動生成

#### 3.3 テンプレート機能
- **提案**:
  - 記事テンプレートの保存・読み込み
  - ブロックのプリセット
  - よく使う構成のパターン化

#### 3.4 高度なテーブル編集
- **現状の問題**:
  - セルの結合が横方向のみ
  - テーブルのネストができない
  - 列の幅調整が直感的でない
- **改善案**:
  - 縦横のセル結合を改善
  - ドラッグで列幅調整
  - テーブルのスタイルプリセット

### 4. コード品質関連

#### 4.1 関心の分離
- **問題**: 1352 行の 1 コンポーネントに全てのロジックが凝集
- **改善案**:
  ```
  RichEditor.svelte (メインコンテナ)
  ├── Toolbar.svelte (ツールバー)
  ├── InlineToolbar.svelte (インラインツールバー)
  ├── SlashMenu.svelte (スラッシュメニュー)
  ├── ColorPicker.svelte (カラーピッカー)
  ├── ElementOverlay.svelte (要素選択枠)
  ├── ResizeHandles.svelte (リサイズハンドル)
  ├── TableMenu.svelte (テーブルメニュー)
  ├── MediaDialog.svelte (メディアダイアログ)
  └── hooks/
      ├── useHistory.ts (履歴管理)
      ├── useAutoSave.ts (自動保存)
      ├── useSelection.ts (選択範囲管理)
      ├── useKeyboard.ts (キーボードショートカット)
      └── useMediaUpload.ts (メディアアップロード)
  ```

#### 4.2 テスト容易性
- **問題**: UI ロジックがコンポーネントに密結合
- **改善案**:
  - カスタムストアの使用
  - ロジックを純粋な関数として抽出
  - Vitest によるユニットテスト追加

#### 4.3 型安全性
- **問題**: `any` 型の使用が散見される
- **改善案**:
  - 厳密な型定義の追加
  - Editor.js の型定義をインポート
  - 型推論の活用

### 5. データ管理

#### 5.1 自動保存の改善
- **現状**: 30 秒ごとにローカルストレージに保存
- **改善案**:
  - 変更検知時の即時保存（debounce あり）
  - サーバーへのバックアップ保存
  - 保存状態のインジケーター表示
  - 競合検出と解決機能の強化

#### 5.2 画像・メディア管理
- **問題**:
  - 画像の遅延読み込みがない
  - 画像の最適化（リサイズ、圧縮）がない
  - 動画のサムネイル生成がない
- **改善案**:
  - 画像の lazy loading 実装
  - クライアント側での画像圧縮
  - サムネイル自動生成

### 6. 拡張性

#### 6.1 プラグインシステム
- **提案**: カスタムブロックの登録機構
  ```typescript
  interface EditorPlugin {
    name: string;
    icon: any;
    render: () => HTMLElement;
    onSave: () => any;
  }
  
  // ユーザーがカスタムブロックを追加可能に
  ```

#### 6.2 エクスポート機能
- **現状**: Markdown エクスポートのみ
- **改善案**:
  - PDF エクスポート
  - Word 形式エクスポート
  - HTML テーマのエクスポート

### 7. セキュリティ

#### 7.1 XSS 対策
- **現状**: `sanitizeHtml` を使用
- **改善案**:
  - CSP ヘッダーの強化
  - 許可タグのホワイトリスト設定を管理画面から
  - 埋め込みコンテンツのサンドボックス化

### 8. 分析・デバッグ

#### 8.1 使用状況分析
- **提案**:
  - よく使う機能のトラッキング
  - 編集時間の計測
  - エラーログの収集

#### 8.2 デバッグモード
- **提案**:
  - 開発モードでの詳細ログ出力
  - 状態のインスペクター
  - パフォーマンスプロファイリング
