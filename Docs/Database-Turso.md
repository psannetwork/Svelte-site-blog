# 🗄️ データベース (Turso) の設定ガイド

このブログでは、データの保存先として **Turso** というクラウドサービスを利用します。これにより、Render の無料プランを使っても、ブログの記事が消えずに残り続けます。

---

## ❓ なぜ Turso を使うのか？

通常、ブログのデータはサーバー内のファイル（`blog.db`）に保存されます。しかし、Render の無料プランでは：

- サイトを更新（デプロイ）するたびに、そのファイルが削除されてしまいます。
- ファイルが消えると、書いた記事や設定がすべて真っさらになってしまいます。

**Turso を使うと、データがサーバーの外にある強力なデータベースに保存されるため、何回サイトを更新してもデータが消えることはありません。**

---

## 🛠️ セットアップ手順

1.  **Turso にログイン**
    [Turso.tech](https://turso.tech/) にアクセスし、GitHub などのアカウントでログインします。

2.  **データベースを作成**
    - **[Create Database]** をクリック。
    - **Name**: `psan-blog-db` など（分かりやすい名前でOK）。
    - **Location**: `Tokyo, Japan (nrt)` を選択。

3.  **URL をコピー**
    画面に表示される **URL** (例: `libsql://your-db-name.turso.io`) をコピーします。これが Render の `TURSO_DB_URL` になります。

4.  **認証トークン (Auth Token) を取得**
    - 画面上の **[Generate Token]** ボタンをクリック。
    - 表示される非常に長い文字列をコピーします。
    - これが Render の `TURSO_DB_AUTH_TOKEN` になります。

---

## ⚠️ 注意：よくある間違い

- **URL の間違い**: 必ず `libsql://` で始まる URL を使ってください。
- **トークンの間違い**: トークンは `eyJhbGc...` で始まる非常に長い文字列です。URL と間違えないように注意してください。
- **有効期限**: トークンには有効期限がない「Default」を選択してください。

---

## 💻 ローカルで Turso を使う場合

自分の PC の `.env` ファイルに、Render に設定したものと同じ URL と Token を書けば、自分の PC からクラウド上の本番データを見ることも可能です。
