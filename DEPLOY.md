# 🚀 Svelte Site Blog デプロイガイド

このブログをインターネットに公開する方法を説明します。
Render.com の無料枠と Turso を組み合わせることで、**完全無料**で自分だけのブログを運営できます。

---

## 📋 事前準備

1.  **GitHub アカウント**: コードを保存・公開するために必要です。
2.  **Render アカウント**: サイトをホスティングするために使用します ([render.com](https://render.com/))。
3.  **Turso アカウント**: データベース（データを保存する場所）として使用します ([turso.tech](https://turso.tech/))。

---

## ☁️ ステップ1: Turso でデータベースを作成する (無料)

Render の無料プランではデータの保存（Disk）が有料なため、データベースは外部の Turso を使うのがおすすめです。

1.  [Turso](https://turso.tech/) にログインします。
2.  **[Create Database]** をクリックし、適当な名前（例: `my-blog-db`）を付けて作成します。
3.  表示された **URL** (例: `libsql://...`) をメモします。
4.  **[Generate Token]** をクリックして **Auth Token** を取得し、メモします。

---

## ☁️ ステップ2: Render.com でサイトを公開する (無料)

1.  **コードを GitHub に Push する**
    あなたの GitHub リポジトリにこのプロジェクトの最新コードをアップロードしてください。

2.  **Render で Web Service を作成**
    - Renderにログインし、**[New +]** → **[Web Service]** を選択。
    - GitHub リポジトリを連携します。

3.  **基本設定**
    - **Name**: 好きな名前（サイトのURLになります）。
    - **Region**: `Singapore` を推奨。
    - **Runtime**: `Docker` (自動で選択されます)。

4.  **環境変数（Environment Variables）を設定**
    **[Advanced]** をクリックし、以下の3つを必ず設定します。
    - `TURSO_DB_URL`: メモした Turso の URL
    - `TURSO_DB_AUTH_TOKEN`: メモした Turso の Auth Token
    - `NODE_ENV`: `production`

5.  **デプロイ開始**
    **[Create Web Service]** をクリックします。5分ほど待って「Service is live 🎉」と表示されれば完了です！

---

## 📝 運用のヒント

- **更新方法**: コードを修正して GitHub に `git push` すれば、Renderが自動的に新しい内容を読み込んでサイトを更新してくれます。
- **バックアップ**: Turso の管理画面から、データベースのバックアップや状態をいつでも確認できます。
- **デザイン変更**: 管理画面の「設定」から、サイト名、アイコン、そしてカスタムCSSを使って見た目を自由に変えられます。

---

## 💻 (参考) 自分のパソコンで動かす方法

開発やテストのためにローカルで動かす手順です。

1.  `bash setup.sh` を実行して初期設定。
2.  `pnpm run dev` で起動。
3.  `http://localhost:5173` にアクセス。

ローカルでは `blog.db` というファイルが自動作成され、そこにデータが保存されます。

## 🛠 トラブルシューティング

- **サイトが重い**: Renderの無料プランはしばらくアクセスがないと「スリープ」します。最初のアクセスに 30秒〜1分 かかることがありますが、仕様です。
- **ログインできない**: Turso の Token が間違っていないか、環境変数の設定を再確認してください。

ブログ運営を楽しんでください！✨
