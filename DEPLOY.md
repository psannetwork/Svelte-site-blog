# 🚀 Svelte Site Blog デプロイガイド

このガイドでは、このブログシステムを自分のパソコン（ローカル環境）で動かす方法と、インターネット（Render.com）に公開する方法を、初心者の方にも分かりやすく説明します。

---

## 📋 事前準備

始める前に、以下のツールがあなたのパソコンにインストールされているか確認してください。

1.  **Node.js (v20以上)**: サイトを動かすための基盤です。[公式サイト](https://nodejs.org/)からダウンロードできます。
2.  **Git**: プログラムのコードを管理するツールです。
3.  **pnpm**: 動作が速いパッケージ管理ツールです（`npm install -g pnpm` でインストールできます）。

---

## 💻 1. 自分のパソコンで動かす（ローカル環境）

開発やテストのために、まずは自分のパソコンで動かしてみましょう。

### 手順

1.  **リポジトリをコピーする**
    ```bash
    git clone <あなたのリポジトリのURL>
    cd Svelte-site-blog
    ```

2.  **セットアップを実行する**
    便利な自動セットアップスクリプトを用意しています。これを実行するだけで、必要なライブラリのインストールと設定ファイルの作成が完了します。
    ```bash
    bash setup.sh
    ```

3.  **開発サーバーを起動する**
    ```bash
    pnpm run dev
    ```

4.  **ブラウザで確認する**
    ターミナルに表示されたURL（通常は `http://localhost:5173`）を開いてください。

💡 **ヒント**: 最初にサイトを開くと「管理者セットアップ」の画面が表示されます。指示に従って最初の管理ユーザーを作成してください。

---

## ☁️ 2. インターネットに公開する（Render.com）

[Render.com](https://render.com/) を使って、無料でサイトを公開する方法です。このブログは **Docker** という仕組みを使って、本番環境でも安定して動くように設計されています。

### 手順

1.  **GitHub にコードをアップロードする**
    あなたの GitHub リポジトリに最新のコードを `push` しておきます。

2.  **Render にログインして新規作成**
    *   Renderにログインし、**[New +]** ボタンから **[Web Service]** を選択します。
    *   GitHub リポジトリを選択（Connect）します。

3.  **基本設定を入力する**
    *   **Name**: 好きな名前（URLの一部になります）。
    *   **Language**: `Docker` を選択（自動で判定されるはずです）。
    *   **Region**: 日本から近い `Singapore` を推奨します。

4.  **【重要】データの保存場所（Disk）を設定する**
    Renderの無料プランでは、再起動するたびにデータベースの中身が消えてしまいます。これを防ぐために「Disk」を設定する必要があります。
    *   設定画面の **[Disks]** セクションで **[Add Disk]** をクリックします。
    *   **Name**: `blog-data`
    *   **Mount Path**: `/app/data`
    *   **Size**: `1GB`（無料で十分な量です）

5.  **環境変数（Environment Variables）を設定する**
    **[Environment]** セクションで以下の2つを追加します。
    *   `DB_PATH`: `/app/data/blog.db`
    *   `NODE_ENV`: `production`

6.  **デプロイ！**
    一番下の **[Create Web Service]** をクリックします。数分待つと「Service is live 🎉」と表示され、発行されたURLから世界中であなたのブログが見れるようになります。

## 💾 3. データベースに Turso を使う (高度な公開方法)

Render の「Disk」を使わずに、外部データベースサービスの [Turso](https://turso.tech/) を使うこともできます。この方法なら、Render の無料プランでもデータの永続化がより簡単になります。

### 手順

1.  **Turso でデータベースを作成**
    *   Turso にサインアップし、新しいデータベースを作成します。
    *   `URL` (例: `libsql://...`) と `Auth Token` を取得します。

2.  **環境変数を設定**
    Render の [Environment] 設定で、以下の2つを追加します。これらを設定すると、自動的に Turso への接続に切り替わります。
    *   `TURSO_DB_URL`: Turso のデータベース URL
    *   `TURSO_DB_AUTH_TOKEN`: Turso の認証トークン

3.  **Disk 設定は不要**
    Turso を使う場合は、Render の **[Disks] 設定は不要** です。

---

## 📝 運用のヒント

*   **更新したいとき**: 自分のパソコンで修正して GitHub に `git push` すれば、Renderが自動的に新しい内容を読み込んでサイトを更新してくれます。
*   **バックアップ**: 管理画面の「設定」から、いつでもデータベースのバックアップをダウンロードできます。定期的に手元に保存しておくと安心です。
*   **デザイン変更**: 管理画面の「設定」から、サイト名、アイコン、そしてカスタムCSSを使って見た目を自由に変えられます。

---

## 🛠 トラブルシューティング

*   **画像が表示されない**: Renderの「Disk」設定が正しいか、マウントパスが `/app/data` になっているか確認してください。
*   **サイトが重い**: 無料プランはしばらくアクセスがないと「スリープ」状態になります。最初のアクセスに数十秒かかることがありますが、故障ではありません。
